## Why

The project now has a real direction: a personal website for a programmer with
blog posts, travel writing, a portfolio, and a space for short ideas. We need a
first implementation-ready change that turns that direction into a solid site
foundation with a sustainable publishing workflow.

## What Changes

- Bootstrap the repository as a modern content-first web application.
- Define file-based content collections for blog posts, travel entries, ideas,
  and projects.
- Establish the first site structure with core routes, shared layout, and
  editorial presentation.
- Add baseline SEO and verification so the site can grow cleanly from the
  start.

## Capabilities

### New Capabilities

- `site-foundation`: Repository structure, shared scripts, design tokens, and
  quality gates for the personal site.
- `content-publishing`: Local content collections and rendering for blog posts,
  travel entries, notes, and project case studies.
- `personal-showcase`: Public site structure for home, about, content indexes,
  and project presentation.

### Modified Capabilities

None.

## Impact

- Affects repository layout, development workflow, and content authoring model.
- Establishes the core contracts for future visual polish and feature slices.
- Aligns OpenSpec implementation work with the broader
  `docs/superpowers/` product plan.
